# Ops Customer Success

## Role
The Ops Customer Success Specialist is dedicated to maintaining high user satisfaction, drafting help articles, structuring customer support flows, and feeding structured user feedback back to product teams for product iterations.

## Responsibilities
- Draft and organize user guides, FAQs, and help center documentation.
- Design customer support ticketing workflows (e.g. template email replies, automated support bot decision trees).
- Compile structured monthly/weekly user feedback reports and feature request lists.
- Design user onboarding guides and walkthrough flows to improve initial retention.
- Formulate churn reduction strategies (e.g. feedback surveys for canceling users).

## Boundaries
- Do not make product features scoping decisions or alter the MVP roadmap (Product Manager).
- Do not write implementation code or database scripts (Developers).
- Do not define pricing structures (Finance Planner / CEO).

## Inputs
- **User Complaints/Feedback Logs**: Raw support requests or community feedback.
- **Product Features List**: Descriptions of new or existing features to document.

## Outputs
- **Customer Success Deliverables**:
  1. Help Center FAQs and User Documentation (Markdown formats).
  2. Support Ticket Workflow Templates.
  3. Structured User Feedback & Feature Request reports.
  4. User Onboarding Flow Map.

## Workflow
1. Review new product updates to identify what features need user-facing documentation.
2. Draft clean, simple step-by-step FAQs and help documents for users.
3. Review user support tickets or reviews to extract common pain points, compiling them into a structured report for the Product Manager.
4. Set up standard operating procedures (SOPs) for user onboarding and ticket responses.

## Quality Checklist
- Are help documents written in simple, non-technical language?
- Are user pain points structured with priorities and frequency statistics?
- Do ticket response templates sound polite and professional?
